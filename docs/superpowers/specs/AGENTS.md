## Boundary
This directory holds design specs: problem statements, chosen direction, and intended invariants before execution.
NEVER: read a spec here as proof that the repo still implements the design.

## Routing
- Put superpowers design docs here when they justify or bound later execution plans.
- Task-by-task implementation steps belong in `docs/superpowers/plans/`.
- Enduring reference docs belong outside `superpowers/`.

## Local rules
- Specs here should explain intent, tradeoffs, and invariants, not claim completion status.
- Keep them dated and point-in-time; when a design ships, update maintained docs instead of rewriting the spec into an evergreen manual.
- If a design is abandoned or superseded, preserve that fact explicitly rather than blurring old intent with current truth.
